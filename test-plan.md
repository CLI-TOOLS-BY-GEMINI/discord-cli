# 実環境テスト (E2E) 計画書

## 目的
Discord API CLI ツールを可能な限り実環境（Discordの実サーバー）でテストするための手法を計画し、自動化するためのガイドラインですわ。

## 実環境テストの計画内容

実環境の Discord API を用いた End-to-End (E2E) テストを安全かつ確実に行うため、以下の手法を採用いたします。

### 1. テスト用環境変数の導入
誤って本番用Botや意図しないサーバーに対してテストを実行しないよう、専用の環境変数が設定されている場合のみ実環境テストが走る仕組みといたしますわ。
- `DISCORD_E2E_TOKEN`: テスト専用のBotトークン
- `DISCORD_E2E_GUILD_ID`: テスト操作が許可されたサーバー(Guild)ID
- `DISCORD_E2E_CHANNEL_ID`: テスト用のメッセージ送信や名前変更を行うチャンネルID

### 2. Go の `testing` パッケージを用いた CLI バイナリテスト (`e2e_test.go`)
- シェルスクリプトではなく Go の標準テスト機構に組み込むことで、`go test` コマンドで統一的にテストを実行できるようにいたします。
- テストの初期化処理 (`TestMain` または各テスト関数内) で `go build -o test-discord-cli ./cmd/discord-cli/main.go` を実行し、最新のバイナリをビルドいたしますわ。
- 各テストケースでは `os/exec` パッケージを用いて、ビルドしたバイナリに実引数を渡して実行し、終了コード (Exit Code) や標準出力の内容を検証いたします。
- `DISCORD_E2E_TOKEN` などの必須環境変数が未設定の場合は、`t.Skip("E2Eテスト用の環境変数が未設定のためスキップいたしますわ")` を呼び出し、通常のCIパイプラインやローカルテストが失敗しないように保護いたしますわ。

### 3. 推奨するテストシナリオ
1. **認証と基本情報の取得**:
   `./test-discord-cli me` および `./test-discord-cli me-guilds` を実行し、APIとの認証が通り情報が取得できるかを確認いたします。
2. **メッセージ送信の検証**:
   `./test-discord-cli message <CHANNEL_ID> "E2Eテストのメッセージですわ"` を実行し、エラーなく終了するかを確認いたします。
3. **チャンネル更新の検証**:
   `./test-discord-cli modify-channel <CHANNEL_ID> "テスト用チャンネル"` を実行し、チャンネル名の変更処理が正常にAPIに受理されるかを確認いたします。
